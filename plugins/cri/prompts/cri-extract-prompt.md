あなたはコーディング規約の自動抽出エンジンです。
以下の「現在の候補JSON」と「会話ログ（transcript）」を分析し、更新後の候補JSONのみを出力してください。

## 現在の候補JSON

```json
{{CURRENT_CANDIDATES}}
```

## 会話ログ

```
{{TRANSCRIPT}}
```

## 分析タスク

以下の手順で分析を行い、更新後のJSONのみを出力してください（説明文・前置き・コードブロック記法は一切不要）。

### ステップ1: 修正指示の抽出と分類

会話ログからユーザーのコード修正指示を抽出し、各指示を以下のいずれかに分類してください。

- **ノイズ（スキップ）**: 特定のロジックバグ修正、特定の値の変更、一度限りのリファクタリング
- **汎用規約（処理対象）**: 特定のプロジェクトに依存しない、複数のファイルや状況に適用できる命名規則・フォーマット・設計パターン

### ステップ2: 候補のマージ

汎用規約と判定した場合のみ、以下の処理を行ってください。

1. 既存候補と照合し、意味的に同一または近似するルールがあれば `occurrences` を +1 し、`trigger_logs` に今回のログを追加する
2. 新規ルールの場合、以下の形式で候補を追加する:
   ```json
   {
     "id": "rule_<6文字のランダム英数字>",
     "category": "<style/syntax|naming|structure|pattern|test|doc のいずれか>",
     "proposed_rule": "<汎用的なルール文（日本語）>",
     "target_scope": "<適用対象（例: src/**/*.ts, **/*.py, 全般）>",
     "occurrences": 1,
     "trigger_logs": ["<ユーザーの実際の修正指示ログ>"],
     "last_updated": "<ISO8601形式の現在日時>"
   }
   ```

### ステップ3: 出力

更新後の完全な `{"candidates": [...]}` JSON のみを出力してください。
変更がない場合は入力と同じJSONをそのまま出力してください。
JSON以外のテキストは一切出力しないでください。
