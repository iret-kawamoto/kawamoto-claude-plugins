# Continuous Rule Improvement (CRI) レビュー実行手順

あなたは有能なリードエンジニアとして、コーディングルールのレビューと統合を対話的に行ってください。以下の手順に従い、ファイル操作ツール（Read, Edit 等）と `AskUserQuestion` を駆使して自律的に作業を進めてください。

## 1. 候補の読み込みと分析

`.claude/tmp/cri-candidates.json` を読み込み、`occurrences >= 3` のルール候補をフィルタしてください。該当がない場合は「✨ 現在レビュー待ちのルールはありません」と伝えて処理を終了してください。

候補がある場合は、既存の `.claude/rules/` 配下のファイルと照らし合わせ、各候補について競合や矛盾がないか分析してください。

## 2. 一括レビュー（AskUserQuestion による複数選択）

分析が完了したら、`AskUserQuestion` の `multiSelect: true` を使って全候補を一括で提示してください。ユーザーが選択した候補だけをマージします。

各選択肢（option）の構成:
- **label**: ルールの要約（短く。例: "PascalCase 命名規則"）
- **description**: 以下を含める
  - 対象スコープ（target_scope）
  - 出現回数（occurrences）
  - 競合チェック結果（競合なし / 既存ルール○○と矛盾あり）
  - 改善提案があればその旨

質問文の例: 「以下のルール候補のうち、`.claude/rules/` に追加するものを選択してください。」

ユーザーが選択肢のいずれかに notes（修正指示）を付けた場合は、その通りに文面を修正してからマージしてください。

## 3. マージ実行

ユーザーが選択した候補について、`.claude/rules/` 内の適切な Markdown ファイルと見出しを特定し、自然な箇条書きとして追記（Edit ツール）してください。

適切なファイルが存在しない場合は、カテゴリに基づいて新規ファイルを作成してください。

## 4. クリーンアップ

処理した候補（マージしたものもスキップしたものも）を `.claude/tmp/cri-candidates.json` から削除し、完了を報告してください。マージした候補の一覧と、追記先のファイルパスを明示してください。
