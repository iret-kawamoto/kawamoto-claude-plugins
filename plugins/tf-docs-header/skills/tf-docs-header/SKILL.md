# terraform-docs ヘッダ付与・更新

編集した Terraform ディレクトリの `main.tf` に terraform-docs 用ヘッダを付与または最新化する。

## ステップ 1: 対象ディレクトリを特定する

直近の編集対象ディレクトリを特定する。**他のディレクトリは対象外**。

```bash
git diff --name-only HEAD
git status --short
```

`.tf` ファイルのパスから `environments/<env>/` または `modules/<module>/` を特定する。
複数ディレクトリが該当する場合はすべてを対象とする。

---

## ステップ 2: 現状を確認してモードを決定する

各ディレクトリの `main.tf` 先頭を確認し、以下のモードを選択する。

```bash
head -5 <対象ディレクトリ>/main.tf
```

| 状態 | モード |
|------|--------|
| `/**` で始まらない | **追加モード**: ヘッダを新規作成して付与 |
| `/**` で始まる | **更新モード**: 既存ヘッダと現在のコードを照合して差分を反映 |

---

## ステップ 3: モジュール情報を収集する

**追加モード・更新モード共通**。ヘッダ内容の根拠を確認する。

- **`variables.tf`**: 変数名・`description`・`required` 状態
- **`outputs.tf`**: 出力値（存在する場合）
- **`main.tf`**: `resource "aws_..."` や `module "..."` ブロック

推測や将来の機能を書かない。**コードに存在するものだけをヘッダに反映する**。

---

## ステップ 4: ヘッダを生成・修正する

### ヘッダフォーマット

#### modules/<module>/main.tf 用

```
/**
 * ## <モジュール名>
 * <モジュールの一文説明>
 *
 * ### Features
 * - <機能1>: <説明>
 * - <機能2>: <説明>
 *
 * ### Usage
 * ```
 * module "<module_name>" {
 *   source = "../../modules/<module_name>"
 *
 *   <必須変数1> = "<値の例>"
 *   <必須変数2> = "<値の例>"
 * }
 * ```
 */
```

**ルール:**
- `### Features` はリソースから判断できる機能のみ列挙
- `### Usage` は `default` なし（必須）の変数のみ記載
- `source` は `../../modules/<module_name>` 形式（`environments/` からの相対パス）
- `### Features` に列挙すべき機能がない場合はセクションごと省略する

#### environments/<env>/main.tf 用

```
/**
 * ## <環境名> 環境
 * <環境の用途・概要>
 *
 * ### 使用モジュール
 * - <モジュール1>: <用途>
 * - <モジュール2>: <用途>
 */
```

**ルール:**
- `### 使用モジュール` は `main.tf` 内の `module "..."` ブロックをすべて列挙する

### 更新モードの差分判断基準

既存ヘッダと現在のコードを照合し、以下のいずれかに該当する場合は更新する。

**内容の乖離:**

| 変更内容 | 更新対象 |
|----------|----------|
| 変数の追加・削除 | `### Usage` の引数一覧 |
| リソース種別の追加・削除 | `### Features` の項目 |
| `module "..."` の追加・削除 | `### 使用モジュール` の項目 |
| モジュールの説明文が実態と乖離している | 説明文 |

**構造の不足（フォーマットルール違反）:**

| 欠けているセクション | 条件 | 対応 |
|---|---|---|
| `### Features` | `modules/` かつ列挙できる機能がある | セクションを追加する |
| `### Usage` | `modules/` かつ必須変数がある | セクションを追加する |
| `### 使用モジュール` | `environments/` かつ `module "..."` ブロックがある | セクションを追加する |

内容の乖離も構造の不足もなければ既存ヘッダをそのまま維持する（不要な書き換えをしない）。

---

## ステップ 5: ファイルを編集する

### 追加モード

`main.tf` の先頭に生成したヘッダを挿入する。Edit ツールで `old_string` を既存の先頭行にして挿入する。

### 更新モード

既存の `/** ... */` ブロック全体を新しいヘッダで置換する。Edit ツールで `old_string` に既存ヘッダ全体を指定して置換する。

**ファイル全体を書き直さない。ヘッダ部分のみを編集する。**

---

## 完了報告

- 付与・更新したディレクトリと変更内容の要旨を報告する
- 更新モードで変更なしと判断したディレクトリはその旨を明示する
