# セキュリティ監査スキル

リポジトリの言語/フレームワークを自動検出し、OWASP Top 10 コード分析・CVE 調査・外部セキュリティツール実行を並列で行い、統一フォーマットのレポートを出力する。過去に「問題なし」と判断された指摘は無視リストで管理し、次回以降スキップする。

## ステップ 1: 対象スコープを確認する

引数からスコープを決定する。

- 引数なし → リポジトリ全体をスコープとする
- ディレクトリパス指定 → そのディレクトリをスコープとする
- `--changed` → `git diff --name-only HEAD` の変更ファイルが含まれるディレクトリをスコープとする

スコープが不明確な場合は AskUserQuestion で確認する：

```
対象スコープを選択してください:
- リポジトリ全体
- 特定のディレクトリ（パスを入力）
- 変更ファイルのみ（git diff）
```

スコープが決まったら、対象ディレクトリの絶対パスを `<TARGET>` として以降のステップで使用する。

## ステップ 2: 無視リストを読み込む

`.claude/memory/security-audit-ignore.md` を Read ツールで読み込む。ファイルが存在しない場合は空の無視リストとして扱う（エラーにしない）。

読み込んだ無視リストの内容を `<IGNORE_LIST>` として保持する。

## ステップ 3: 3つのエージェントを並列で起動する

以下の3つのエージェントを **1つのメッセージで同時に** Agent ツールで起動する。全て完了するまで待機する。

### 3a: security-audit-scanner

```
subagent_type: security-audit-scanner
model: sonnet
prompt: "対象ディレクトリ <TARGET> のセキュリティスキャンを実行してください。"
```

### 3b: security-audit-owasp-analyzer

```
subagent_type: security-audit-owasp-analyzer
model: sonnet
prompt: "対象ディレクトリ <TARGET> の OWASP Top 10 分析を実行してください。"
```

### 3c: security-audit-cve-checker

```
subagent_type: security-audit-cve-checker
model: sonnet
prompt: "対象ディレクトリ <TARGET> の依存関係 CVE チェックを実行してください。"
```

いずれかのエージェントが失敗した場合、その結果を `[ERROR: <エラー内容>]` として記録し、他のエージェントの結果は正常に処理する。

## ステップ 4: reporter エージェントでレポートを統合する

3つのエージェントの出力が揃ったら、security-audit-reporter エージェントを起動する。

```
subagent_type: security-audit-reporter
model: sonnet
prompt: |
  以下の3つのスキャン結果と無視リストを統合し、セキュリティ監査レポートを生成してください。

  ## スキャン対象
  <TARGET>

  ## スキャナー結果
  <ステップ3aの出力をそのまま貼付>

  ## OWASP 分析結果
  <ステップ3bの出力をそのまま貼付>

  ## CVE チェック結果
  <ステップ3cの出力をそのまま貼付>

  ## 無視リスト
  <IGNORE_LIST の内容、または「(なし)」>
```

## ステップ 5: レポート表示と無視リスト更新

reporter の出力をそのままユーザーに表示する。

指摘が1件以上ある場合、AskUserQuestion で確認する：

```
「問題なし」として無視リストに登録したい指摘はありますか？
- はい（指摘IDをカンマ区切りで入力してください）
- いいえ（このまま終了）
```

ユーザーが指摘IDを指定した場合：

1. 各指摘IDについて無視理由を AskUserQuestion で確認する
2. `.claude/memory/security-audit-ignore.md` が存在しなければ以下のテンプレートで新規作成する：

```markdown
---
description: セキュリティ監査スキルの無視リスト。「問題なし」と判断された指摘を記録する。
updated: <今日の日付 YYYY-MM-DD>
---

# セキュリティ監査 無視リスト

## エントリ
```

3. 指定された指摘を以下の形式で追記する：

```markdown
### <ID>
- **カテゴリ**: <IaC / OWASP / CVE / Secret>
- **検出元**: <検出元>
- **ファイル**: <ファイルパス>
- **説明**: <指摘内容>
- **無視理由**: <ユーザーが入力した理由>
- **登録日**: <今日の日付 YYYY-MM-DD>
```

4. frontmatter の `updated` を今日の日付に更新する

## 制約事項

- コードの自動修正は行わない（レポートのみ）
- 外部ツール（checkov/trivy/gitleaks）は存在確認してから実行する（未インストール時はスキップ）
- CVE の WebSearch は上限20件のパッケージに対して実施する
- 無視リストはプロジェクトスコープ（`.claude/memory/`）に保存する
