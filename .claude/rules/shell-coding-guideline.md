# シェルスクリプト コーディングガイドライン（要約）

> Google Shell Style Guide に基づく。詳細な例や解説が必要な場合は `shell-guideline-lookup` エージェントを使用すること。
> 完全版: `.claude/docs/shell-coding-guideline.md`

## 適用範囲
- シェルスクリプトは小規模ユーティリティ（目安: 100行以下）に限定する
- 100行超 or 複雑な制御フローが必要な場合は Python 等へ移行する

## ファイルヘッダ・インタープリタ
- シバン: `#!/usr/bin/env bash`
- 先頭に `set -euo pipefail` を設定する
- ファイル冒頭コメントに目的・使用法を記載する
- 拡張子は `.sh`。SUID/SGID は禁止

## 出力
- 正常出力は STDOUT、エラー・診断は STDERR（`>&2`）
- エラー出力関数の定義を推奨: `err() { echo "[$(date +'%Y-%m-%dT%H:%M:%S%z')]: $*" >&2; }`

## コメント
- ファイルヘッダ: 概要と使用法を記述する
- 関数コメント: Globals / Arguments / Outputs / Returns を記載する
- TODO: `# TODO(username): 説明` の形式で記述する

## フォーマット
- インデント: スペース2つ（タブ禁止）
- 行長: 最大80文字
- パイプライン: 1行に収まらない場合、1コマンドずつ改行し `|` を行頭に置く
- `; then` / `; do` は `if` / `for` / `while` と同じ行に記述する
- case: パターンは2スペースインデント。`;&` / `;;&` は禁止

## 変数・クォート
- 変数展開は常に `"${var}"` でクォートする
- `"$@"` を使用する（`$*` は原則禁止）
- リテラル文字列にはシングルクォート、リテラル整数はクォート不要
- 例外: `(( ))` 内と `[[ ]]` 右辺の正規表現

## 機能・構文
- **ShellCheck** による静的解析を必須とする
- コマンド置換は `$(command)`（バッククォート禁止）
- テスト構文は `[[ ]]`（`[ ]` / `test` 禁止）
- 空文字列チェック: `-z` / `-n` を明示的に使用する
- 数値比較: `(( ))` を使用する（`[[ ]]` 内の `>` / `<` は辞書順のため禁止）
- ワイルドカード展開: `rm ./*` のように明示的パスを指定する
- **`eval` は使用禁止**
- リスト管理には配列を使用する（文字列でリストを管理しない）
- パイプ → while: プロセス置換 `< <(cmd)` または `readarray` を使用する
- 算術演算: `$(( ))` / `(( ))`（`let` / `$[ ]` / `expr` 禁止）
- スクリプト内の `alias` は禁止（関数を使用する）

## 命名規則
- 関数名: `lower_snake_case`（パッケージ区切り `::` 可）
- 変数名: `lower_snake_case`
- 定数・環境変数: `UPPER_SNAKE_CASE` + `readonly` で宣言
- ローカル変数: `local` で宣言必須。宣言と代入は分離する（コマンド置換時）

## 構造
- 関数はファイル上部（定数宣言の直後）にまとめて配置する
- 他の関数を含むスクリプトには `main` 関数を定義し、末尾で `main "$@"` を呼び出す

## コマンド実行
- 戻り値は即座にチェックする（`if !` パターン or `|| return`）
- パイプラインは `PIPESTATUS` で確認する
- ビルトインを外部コマンドより優先する（`$(( ))` > `expr`、パラメータ展開 > `sed`）
